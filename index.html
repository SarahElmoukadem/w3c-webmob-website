How We Work

Define a Problem through Use Cases

There are some things that native apps can do that the web can't. We identify these things and then document the use cases and requirements. These use cases go on to support new standards on the web platform.

Example: Installable Web Apps

Github

We do all our work on Github. Github makes it easy to track and log issues and everyone can contribute. Everyone can browse our work on the main Webmob Github Page and start forking repositories and sending in pull requests.

Example: Screen Wake Use Cases
